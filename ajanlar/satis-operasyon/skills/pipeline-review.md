# Skill: Review the Sales Pipeline

## Purpose
Analyze supplied pipeline data and identify evidence-backed conversion bottlenecks.

## When to Use This Skill
- "Satış hunimizi analiz et."
- "Nerede müşteri kaybediyoruz?"

## Do NOT Use This Skill For
- A single-lead qualification -> use `lead-qualification.md`.
- Financial health analysis -> use the Business Strategy agent.

## Process
1. Map the supplied stages and counts.
2. Calculate conversion rates only from supplied data.
3. Identify the largest stage drop and compare wins and losses where data permits.
4. Use `.agents/hey-sales/SALES_STRATEGY.md` as the reporting framework.
5. Provide up to three measurable recommendations and label low-sample observations as emerging.

## Quality Bar
- Patterns from fewer than five observations are marked emerging.
- Recommendations include the metric that will validate them.

## Rules & Feedback
- Do not infer missing funnel stages.
  Why: invented denominators make conversion analysis invalid.
  How to apply: show unavailable metrics as missing.
