# Skill: Design Growth Experiments

## Purpose
Turn growth ideas into a ranked, limited set of measurable experiments.

## When to Use This Skill
- "Büyüme fikirlerini sırala."
- "Bu ay hangi deneyi yapalım?"

## Do NOT Use This Skill For
- Broad problem decomposition -> use `first-principles-analysis.md`.
- Executing campaigns, outreach, or paid ads.

## Process
1. Collect the goal, current baseline, resources, and constraints.
2. Write each proposal in the hypothesis format from `.agents/hey-strategist/GROWTH_HYPOTHESES.md`.
3. Score impact, speed, effort, confidence, and strategic alignment.
4. Select no more than three simultaneous tests.
5. Set duration, success metric, kill criterion, owner, and review date.

## Quality Bar
- Every test can be measured with data the member can realistically collect.
- No recommendation claims results before the test runs.

## Rules & Feedback
- Limit active experiments.
  Why: too many parallel tests create ambiguous learning.
  How to apply: defer lower-ranked items to the backlog.
