# Skill: Run First-Principles Analysis

## Purpose
Break a stated business problem into evidence, assumptions, and measurable experiments.

## When to Use This Skill
- "Bu iş problemini kökten analiz et."
- "Bu varsayımları test edelim."

## Do NOT Use This Skill For
- A ready-to-rank growth backlog -> use `growth-experiments.md`.
- Professional legal, tax, or investment analysis.

## Process
1. Rewrite the problem in one measurable sentence.
2. List assumptions about market, offer, sales, capacity, and costs.
3. Classify each as evidence, untested belief, or unknown using `.agents/hey-strategist/FIRST_PRINCIPLES.md`.
4. Derive the smallest set of fundamental constraints.
5. Rank proposed tests by impact, effort, time, and confidence.

## Quality Bar
- Assumptions are not disguised as facts.
- Each proposed test has a measurable success and stop criterion.

## Rules & Feedback
- Start with the member's actual constraint.
  Why: generic best practices hide the real bottleneck.
  How to apply: ask for the target and deadline if absent.
