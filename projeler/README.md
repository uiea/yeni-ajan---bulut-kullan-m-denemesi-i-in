# Web Site Projects Archive

This is the single local archive for all web-site client work.

## Structure

```text
projeler/
  [client-slug]/
    [project-slug]/
      README.md
      brief.md
      erisimler.md
      teslimler.md
      ... project files
```

## Rules

- Create every project inside its client's folder: `projeler/[client-slug]/[project-slug]/`.
- Add a project only after the client has a confirmed entry in `ajanlar/web-sitesi-yonetimi/musteriler.md`.
- Keep source files, assets, drafts, project notes, and delivery records inside the matching project folder.
- Do not store passwords, tokens, API keys, or private secret values in this archive. Use a secure reference only.
- Never overwrite a file. Save dated or versioned revisions instead.
- External deployment, publishing, uploading, and client delivery require explicit action-specific approval.
