# Skill: Growth Hypotheses Engine

## Purpose
Generate, rank, test, and kill growth hypotheses systematically. Every growth initiative is a hypothesis with clear success criteria and a kill switch.

## Hypothesis Format
```
HYPOTHESIS: If we [action], then [measurable outcome], because [reasoning].
EXPECTED IMPACT: Revenue +/- X, Time to impact, Confidence
EXPERIMENT: Minimum viable test, Duration, Cost, Success metric, Kill criteria
RESULTS: Start date, Data, Outcome, Learning
```

## Categories
1. **Revenue Growth** - pricing, new streams, upsell, new segments, partnerships
2. **Conversion Optimization** - sales process, qualification, objection handling, trust signals
3. **Cost Reduction** - tool consolidation, automation, renegotiation, elimination
4. **Leverage & Scale** - content that sells, referral programs, product-led growth, partnerships

## Ranking Framework (weighted score)
| Factor | Weight |
|--------|--------|
| Expected Revenue Impact | 3x |
| Speed to Impact | 2x |
| Effort Required | 2x (10 = low effort) |
| Confidence in Outcome | 1.5x |
| Strategic Alignment | 1.5x |

Score > 70: DO NOW | 50-70: DO NEXT | 30-50: CONSIDER | < 30: KILL

## Rules
- Maximum 3 hypotheses testing simultaneously
- Review every Friday
- Must kill or validate within 2-4 weeks
- If can't measure it, don't test it
- Never test the same hypothesis twice without new evidence
