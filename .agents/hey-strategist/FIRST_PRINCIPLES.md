# Skill: First Principles Engine

## Purpose
Break any business problem down to fundamental truths. Strip away assumptions, analogies, and "best practices." Rebuild the solution from physics.

## The Method

### Phase 1: Define the Problem (1 sentence)
Bad: "We need more revenue"
Good: "We need to generate $X/month in recurring revenue within 90 days or we run out of runway"

### Phase 2: Surface All Assumptions
List EVERY assumption about market, product, sales, and costs. Be brutal.

### Phase 3: Challenge Each Assumption
1. Is this actually true? Evidence?
2. How do we know? Data, or gut feeling?
3. What if the opposite were true?
4. What's the cost of this assumption being wrong?

### Phase 4: Identify Fundamental Truths
Only things backed by data or physics.

### Phase 5: Reason Up From Truths
```
Truth: Revenue must exceed costs
Truth: Current monthly cost is X
Therefore: We need at least X/month revenue
Truth: Average deal is Y
Therefore: We need X/Y deals per month
... (continue reasoning chain)
```

### Phase 6: Quantify & Rank Solutions
| Lever | Expected Impact | Effort | Time to Impact | Confidence |

### Phase 7: Design the Experiment
- Hypothesis, minimum viable test, success metric, kill criteria, cost of experiment
